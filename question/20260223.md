## 🌟 **練習問題：ログ解析 その2（難易度：中）**

会社のシステムログファイル（テキスト形式）があり、内容は以下のような行で構成されている：

```
2026-02-20 10:10 LOGIN eri
2026-02-20 10:15 LOGOUT eri
2026-02-20 10:20 LOGIN taro
2026-02-20 11:00 LOGIN eri
2026-02-20 11:15 LOGOUT taro
2026-02-20 11:30 LOGOUT eri
```

1 行は

```
日時（yyyy-MM-dd） 時刻（HH:mm） 操作 LOGIN/LOGOUT ユーザー名
```

の4つが並んでいる。

---

## 🎯 **【問1】 ユーザーごとのログイン回数を数え、昇順で出力せよ**

出力例：

```
eri 2
taro 1
```

---

## 🎯 **【問2】 最も「ログイン時間が長かったユーザー」を求めよ**

1回の LOGIN〜LOGOUT を「セッション」とし、
総滞在時間（分）を集計して、一番長いユーザー名と時間を出力せよ。

出力例：

```
eri 85
```

※ LOGIN したのに LOGOUT がないレコードは無視してよい（不完全なセッションは除外）。

---

## 🎯 **【問3（応用）】 その日の平均セッション時間を求めよ**

セッション（LOGIN〜LOGOUT）の **平均時間（分）** を求めて小数点切り捨てで表示せよ。

出力例：

```
average 42
```

---

## 📝 **ヒント**

* `record Log(LocalDate date, LocalTime time, Ope ope, String user)`
* `enum Ope { LOGIN, LOGOUT }`
* `java.time`（`LocalDate`, `LocalTime`, `Duration`）を使うとすごく楽
* groupBy, sorting, mapping、全部 stream で書ける
* セッション計算は
  「ユーザーごと → 時系列にソート → LOGIN の次の LOGOUT を対応付ける」
  の手順

---

## 🌱 **狙い**

この問題で、以下を復習できる：

* record / enum
* Stream API（filter, map, groupBy, flatMap）
* Comparator
* LocalDate/LocalTime/Duration
* Map＜ユーザー, セッション一覧＞
* ループでの「前回の操作」を覚えておくパターン
